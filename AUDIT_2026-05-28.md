### [ ] A-19b — HIGH (bug) — lens batch4
**A-19b deterministic fixer hardcodes InArgument/x:String, corrupts InOut and non-string args**

- **Detalhe:** A-19b (apply_class: deterministic, auto-applied) flags any missing In OR InOut argument declared by the callee — the detector at detect_a19b_in_args_missing collects `required = {n for n,t in props if t.startswith('InArgument') or t.startswith('InOutArgument')}`. The rule description explicitly says it covers `In` and `InOut` (line 2176-2178), so the fixer silently produces a malformed/wrong binding for every case except a string In arg. The post-fix XAML well-formed gate will NOT catch this (the XML is well-formed; only the type/direction is wrong), so a wrong binding can be written to disk.

### [ ] M-2 — HIGH (bug) — lens batch11
**M-2 fixer builds invalid x:TypeArguments for generic/unknown-typed required args (unescaped '<' = malformed XAML; 'System.?')**

- **Detalhe:** The M-2 detector forwards prop_type=a.type verbatim, including schema types that are generics (e.g. CreateList.NewList type 'Collections.Generic.IList<T>', ChangeType.ConvertedValue type 'T') or unknown ('?', e.g. GetRowItem.Value). _format_property_element_value (fixers.py L41-61) only aliases a small fixed set of System.* types; everything else is passed through with a 'System.' prefix prepended. Results: (1) generic 'Collections.Generic.IList<T>' becomes x:TypeArguments="System.Collections.Generic.IList<T>" — the literal '<' inside an XML attribute value is NOT well-formed XML (lxml: "Unescaped '<' not allowed in attributes values"). The safety.py XML well-formed gate then rolls the fix back, so the deterministic fix silently no-ops on a breaking ERROR (required arg never added). (2) unknown type '?' becomes x:TypeArguments="System.?" — well-formed XML but an unresolvable .NET type, so it PERSISTS and causes a Studio load error. Either way the deterministic auto-fix is wrong for generic/unknown-typed required args.
- **Evidencia:** fixers.py L47-61: bare=(prop_type).split('`')[0]; non-System types get 'System.'+bare; alias map misses generics/'?'. Confirmed outputs: fmt('Collections.Generic.IList<T>',None)='<InArgument x:TypeArguments="System.Collections.Generic.IList<T>" />' (lxml: Unescaped '<' not allowed in attributes values); fmt('?',None)='<InArgument x:TypeArguments="System.?" />'.
